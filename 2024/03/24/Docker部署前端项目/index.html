Docker 部署前端项目详细指南

部署流程概览

  1. 在本地开发环境打包前端项目
  2. 在服务器上准备部署环境
  3. 配置 Nginx
  4. 构建和运行 Docker 容器

1. 本地开发环境操作

1.1 打包前端项目

⚠️ 打包说明：

  1. 确保 package.json 中配置了正确的 build 命令
  2. 打包后的文件通常在 dist 或 build 目录
  3. 检查打包后的文件是否完整

4.4 运行容器

⚠️ 运行参数说明：

  1. -d: 后台运行容器
  2. --name: 指定容器名称
  3. -p: 端口映射（主机端口:容器端口）
  4. -v: 挂载数据卷（主机路径:容器路径）

7. 常见问题解决

7.1 404 错误

⚠️ 可能原因：

  1. 文件路径错误
  2. 文件权限问题
  3. 单页应用路由配置错误

7.2 502 Bad Gateway

⚠️ 可能原因：

  1. 容器未正常启动
  2. 端口被占用
  3. 配置文件错误

7.3 静态资源加载失败

⚠️ 解决方案：

  1. 检查文件路径
  2. 确认文件权限
  3. 检查 Nginx 配置

总结

通过以上步骤，我们完成了前端项目的 Docker 部署。主要优势包括：

  • 环境一致性
  • 快速部署
  • 易于维护
  • 配置灵活

建议定期检查容器状态和日志，及时更新配置和优化性能。
